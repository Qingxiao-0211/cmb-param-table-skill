# 招商银行参数表制作 Skill

> KimiClaw / OpenClaw 技能 —— 根据基金公司标准参数表，自动生成招商银行公募基金产品上线导入模板。

## 📌 解决的问题

基金运营同事在制作招商银行公募基金参数表时，需要将基金公司提供的**标准参数表**手动填入招商银行提供的**导入模板（Excel）**。这个过程：

- 字段多（130+个参数）
- 标黄字段每个产品都不同，容易填错
- 标蓝字段需要根据产品类型判断
- 重复性高，耗时费力

本 Skill 实现**一键自动填充**：上传标准参数表 + 招行空模板 → 自动生成填写完整的招行参数表。

## 🎯 使用方法

安装完成后，日常使用非常简单：

### 步骤 1：上传文件

在 KimiClaw 对话中，上传两个文件：
- **标准参数表**（基金公司提供，含所有产品参数的 Excel）
- **招行参数表模板**（招商银行提供的导入模板，B 列基本为空）

### 步骤 2：发送指令

用自然语言描述你的需求即可，例如：

> "根据这份标准参数表，帮我制作招商银行参数表"

> "我有富国基金的标准参数表，需要生成招行的产品上线导入模板"

> "请把标准参数表的数据填到招行模板里"

### 步骤 3：获取结果

KimiClaw 会自动：
1. 读取并解析标准参数表中的所有字段
2. 按照招行模板的格式要求，自动填充 B 列
3. 标黄字段严格从标准表提取
4. 标蓝字段根据产品名称/类型智能判断
5. 白色字段填充标准默认值
6. 生成完整的 Excel 文件供你下载和检查

## ✅ 支持的字段映射（核心能力）

本 Skill 覆盖招行参数表全部 130+ 个字段的自动填充：

| 区域 | 字段数 | 处理方式 |
|------|--------|----------|
| 产品基本信息 | 24 个 | 基金代码、名称、费率、封闭期等严格映射 |
| 结算账户 | 16 个 | 认购/申购账户信息自动关联 |
| 回款来源账户 | 15 个 | 赎回款/分红款/认购退款账户自动关联 |
| 认购参数 | 10 个 | 起止日期、费率、上下限等 |
| 申购/定投 | 10 个 | 费率、单位、上下限等 |
| 赎回/回款 | 10 个 | 赎回费率、资金清算速度等 |
| 基金转换 | 5 个 | 默认参数 |
| 转托管 | 5 个 | 默认参数 |

### 智能处理亮点

- **封闭期提取**：从基金简称自动提取（如"6个月持有期" → 6, M:个月）
- **费率最高档**：自动识别多档费率结构，提取最高档（排除固定金额门槛如"1000元"）
- **基金状态判断**：根据发行日期自动判断是"发行期"还是"交易期"
- **投资地区判断**：FOF→亚太，QDII→根据投资范围，其他→中国大陆
- **账户信息关联**：自动将申购账户映射为赎回款/分红款账户，认购账户映射为认购退款账户
- **日期解析**：自动解析"2026-05-11至2026-05-29"格式的发行日期范围

## 📋 输入文件要求

### 标准参数表

基金公司提供的标准格式，通常包含：
- 第1列：参数名称（如基金代码、基金名称、管理费率等）
- 第2列：参数值
- 账户信息类字段可能分布在第3-5列（分组形式）

支持的基金类型：
- FOF / 基金中基金
- 混合型基金
- 债券型基金
- 货币型基金 / 现金管理
- 股票型基金 / 指数基金
- QDII 基金（全球/亚太/美国/欧洲/日本/拉美等）
- 养老目标基金
- 持有期 / 滚动持有期 / 定期开放产品

### 招行参数表模板

招商银行提供的标准导入模板 Excel，要求：
- A 列参数名称不可修改
- B 列值为空（由本 Skill 自动填充）
- C 列备注保持原样

## 🔍 生成后检查清单

虽然本 Skill 能自动处理绝大多数字段，但以下特殊情况建议人工复核：

| 检查项 | 说明 |
|--------|------|
| 标黄字段 | 重点核对基金代码、名称、费率、账户信息是否正确 |
| 特殊产品 | QDII、货币基金、定开产品等需确认投资地区、显示模式等 |
| 费率结构 | 如果标准参数表费率结构特殊，核对最高档费率 |
| 日期信息 | 募集起止日期、基金成立日期是否准确 |
| 账户信息 | 银行账号、大额支付号是否去除了空格 |

## 💡 常见问题

**Q: 标准参数表格式和示例不完全一样怎么办？**
A: 本 Skill 基于常见标准参数表格式设计。如果字段名称或布局差异较大，可以手动调整标准表使其与示例格式一致，或者联系开发团队更新 Skill。

**Q: 生成的招行参数表可以直接导入银行系统吗？**
A: 生成后建议先人工复核标黄和标蓝字段，确认无误后再提交银行。

**Q: 一个基金有多个份额类别（A类/C类）怎么办？**
A: 每个份额类别需要单独生成一份招行参数表。标准参数表中通常有多列，分别对应不同份额类别的值。

**Q: Skill 一句话安装没反应？**
A: 你的平台可能不支持自动 git clone。请改用「方式二：手动下载安装」，去 Releases 下载 `.skill` 文件后手动上传。

**Q: 安装后找不到 Skill？**
A: 确认安装路径是否正确。不同平台的 skills 目录位置不同：
- OpenClaw: `~/.openclaw/skills/`
- KimiClaw: `~/.kimi/skills/`
- Claude Code: `~/.claude/skills/`
- Codex: `~/.codex/skills/`
