# 招商银行参数表字段映射规则

本文档详细说明标准参数表字段与招行参数表字段之间的映射关系。

## 映射逻辑概述

招行参数表B列分为三类：
- **标黄字段**：每个产品不同，必须严格从标准参数表提取
- **标蓝字段**：大部分产品固定，部分根据产品名称/类型变动
- **白色字段**：基本不变，使用默认值

## 基本信息区域（基金公司提供的产品相关参数）

| 招行参数 | 数据来源 | 说明 |
|---------|---------|------|

| 养老目标基金风险揭示书 | 固定值 | N:否 |
| 是否上传基金产品资料概要 | 固定值 | N:否 |

## 特殊处理说明

### 费率最高档提取逻辑
标准参数表的费率通常分多档，如：
- M＜100万元: 0.50%
- 100万元≤M＜500万元: 0.30%
- 500万元≤M: 1000元

提取逻辑：
1. 包含`%`的值视为费率
2. 纯数字且小于10的值视为费率（如0.5, 1.2）
3. 纯数字大于等于100的值视为固定金额门槛，忽略
4. 取所有费率中的最大值

### 日期范围解析
标准参数表的发行日期格式为：`YYYY-MM-DD至YYYY-MM-DD`
解析为募集开始日期和募集结束日期，格式化为`YYYY-MM-DD 00:00:00`

### 账户信息空格处理
银行账号和支付行号必须去除所有空格，避免导入失败。

### 持有期/封闭期提取
从基金简称中提取：
- 模式：`数字 + 月/个月/日/天/年/季`
- 示例："富国盈和优选6个月持有期混合" → 6, M:个月
- 示例："某基金90天滚动持有" → 90, D:日
